Chapter 10
- This chapter discussed about debugging
- There are 3 types of errors: system error, programmer error, and user error
- An exception is an error that produces a return value that can then be used by the program to deal with the error
- Stack traces are exceptions. It basically works backwards from the error to the start of the sequence
- Strict mode: will increase the chance of errors, but will be beneficial later on to avoid problems
- Just need to use the string 'use strict'
- you can use the alert() method to test debugging
- console is a more modern solution
- this chapter also reviews a lot of math in javascript and that this might help with debugging
- methods to know: toThrow(), isPrime(), alert(), console.log(), console.trace(), etc
- also need to know the try, catch, and finally in javascript
